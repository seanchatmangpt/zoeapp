# Security and Identity in Zoe Framework

The Zoe Framework SDK implements a forward-looking security architecture designed for the 2030 landscape. It moves beyond static credentials toward **Continuous Trust**, **Post-Quantum Resilience**, and **Zero-Knowledge Privacy**.

## 1. Zero-Knowledge Proof (ZKP) Claims (`auth/zkp`)

Zero-Knowledge Proofs allow users to prove specific claims about their identity (e.g., "I am over 18", "I have a valid driver's license") without revealing the underlying sensitive data (e.g., actual birth date, license number).

### Core Concepts
- **Selective Disclosure**: Reveal only what is necessary for the transaction.
- **Privacy-Preserving**: The verifier never sees the private inputs.
- **On-Chain/Off-Chain**: Supports both centralized and decentralized verification.

### Code Example: `useZkClaimVerifier`

```tsx
import { useZkClaimVerifier } from '@zoe/framework/auth/zkp';

export const AgeVerificationComponent = () => {
  const { verifyClaim, isVerifying, result, error } = useZkClaimVerifier();

  const handleVerify = async () => {
    const claim = { 
      id: 'legal-age-check', 
      field: 'age', 
      operator: 'GTE', 
      threshold: 18 
    };
    
    // The proof would typically be generated by the user's wallet/agent
    const proof = { 
      claimId: 'legal-age-check', 
      proofData: '0x...zkp_payload...', 
      publicSignals: ['1'] // 1 represents success for the GTE check
    };

    const outcome = await verifyClaim(claim, proof);
    if (outcome.verified) {
      console.log('User is verified to be over 18!');
    }
  };

  return (
    <div>
      <button onClick={handleVerify} disabled={isVerifying}>
        {isVerifying ? 'Verifying Proof...' : 'Verify Age (ZKP)'}
      </button>
      {result?.verified && <p>Access Granted ✅</p>}
      {error && <p style={{ color: 'red' }}>Verification Failed: {error}</p>}
    </div>
  );
};
```

## 2. Post-Quantum Identity Verification (`2030/identity`)

As quantum computing threatens traditional asymmetric cryptography (RSA, ECC), Zoe introduces **Post-Quantum (PQ) Identity**. This layer uses NIST-standardized algorithms like Dilithium and Falcon to sign identity proofs.

### Innovations
- **PQ-Signatures**: Every identity assertion is signed with quantum-resistant keys.
- **Cryptographic Receipts**: Proof of verification is encapsulated in a `PqReceipt` for non-repudiation.
- **Quantum-Resistant ZKP**: Integrating PQ-crypto within the ZKP verification pipeline via `PostQuantumZkEngine`.

## 3. Continuous Behavioral Biometrics (`auth/behavioral`)

Zoe replaces the "static login" model with **Continuous Authentication**. By monitoring passive behavioral telemetry, the framework maintains a real-time **Trust Score**.

### Behavioral Metrics
- **Typing Rhythm**: Inter-keystroke intervals and patterns.
- **Interaction Frequency**: Navigation speed and rhythm.
- **Device Handling**: Touch pressure and accelerometer patterns (on supported hardware).

### Code Example: `useBehavioralAuth`

```tsx
import { useBehavioralAuth } from '@zoe/framework/auth/behavioral';

export const SecureSessionMonitor = () => {
  // sensitivity: how much trust is lost on anomaly detection
  const { trustScore, metrics, recordInteraction } = useBehavioralAuth({
    updateInterval: 2000,
    sensitivity: 0.8
  });

  return (
    <div onMouseMove={recordInteraction} onTouchStart={recordInteraction}>
      <h3>Session Trust Score: {(trustScore * 100).toFixed(2)}%</h3>
      <div style={{ 
        width: '100%', 
        height: 10, 
        background: trustScore < 0.5 ? 'red' : 'green' 
      }} />
      
      {trustScore < 0.4 && (
        <div className="alert">
          ⚠️ Low trust detected. Re-authentication may be required.
        </div>
      )}
      
      <p>Typing Speed Interval: {metrics.typingSpeed.toFixed(0)}ms</p>
      <p>Interactions/Min: {metrics.navigationRhythm}</p>
    </div>
  );
};
```

## 4. Passwordless Biometric Identity (`auth/biometric`)

Leveraging native hardware capabilities (FaceID, TouchID, Iris scans), Zoe provides a high-level hook for secure, passwordless authentication.

### Code Example: `useBiometricAuth`

```tsx
import { useBiometricAuth } from '@zoe/framework/auth/biometric';

export const BiometricLoginButton = () => {
  const { 
    authenticate, 
    isAuthenticating, 
    hasHardware, 
    isEnrolled 
  } = useBiometricAuth();

  const handleLogin = async () => {
    if (!hasHardware || !isEnrolled) {
      alert('Biometrics not available on this device');
      return;
    }

    const result = await authenticate({
      promptMessage: 'Confirm identity to access secure vault',
      fallbackLabel: 'Use Device Passcode'
    });

    if (result.success) {
      // Proceed to secure area
    }
  };

  return (
    <button onClick={handleLogin} disabled={isAuthenticating}>
      {isAuthenticating ? 'Scanning...' : 'Login with Face/Touch ID'}
    </button>
  );
};
```

## 5. Multi-Factor Authentication (`auth/mfa`)

Zoe supports a flexible MFA lifecycle including TOTP, SMS, Email, and Push-based challenges.

### Key Features
- **MFA Wrappers**: Use `withMfa` to protect specific sensitive functions.
- **Challenge Lifecycle**: Managed state for pending, successful, and expired challenges.

```tsx
const { withMfa } = useMfaVerification();

const handleTransfer = async () => {
  await withMfa(async () => {
    // This code only runs after successful MFA
    await executeBankTransfer(amount);
  }, { strategy: 'totp' });
};
```

## 6. Multi-Tenant Organizational Identity (`auth/identity`)

Designed for enterprise scale, the `IdentityManager` handles complex organizational structures.

- **Hierarchical Groups**: Nest departments and projects.
- **Inherited Permissions**: Permissions flow from parent groups to child groups.
- **Tenant Switching**: Seamlessly switch between different organizational contexts without re-logging.

## 2030 Best Practices

1. **Prefer ZKP over PII**: Whenever possible, verify claims (ZKP) instead of collecting Personally Identifiable Information (PII).
2. **Implement Step-up Auth**: Use `useBehavioralAuth` to monitor trust and trigger `useMfaVerification` only when the trust score drops.
3. **Quantum Readiness**: Always include `PostQuantumZkEngine` in sensitive financial or medical transaction pipelines.
4. **Passive over Active**: Use behavioral biometrics to reduce user friction while increasing security.
